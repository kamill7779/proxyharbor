# ProxyHarbor v0.3.0 计划：轻量级云原生 HA 基线

> 版本定位：v0.3.0 从 v0.2.0 的单实例友好平台模型，推进到可在 Kubernetes 中以 2~5 个副本稳定运行的轻量 HA 基线。
> 核心原则：先保证多实例正确性和可运维性，不把 ProxyHarbor 做成重型分布式控制平面。

---

## 1. 目标

v0.3.0 要让多实例部署具备以下性质：

- 多个实例不会重复执行全局单例后台任务。
- lease 续租、更新、撤销具备基本并发保护。
- cache 失效可以跨实例传播，并以数据库为最终权威来源。
- 健康写入不会因副本数增加而成倍污染统计。
- Kubernetes 滚动升级时实例可以先 drain 再退出。
- 运维人员可以通过 Prometheus 和 Admin API 看见集群状态、leader 状态和配置漂移。

---

## 2. 非目标

v0.3.0 明确不做以下重型能力：

- 不实现 Raft、etcd、Consul 或复杂一致性协议。
- 不实现实时强一致 lease watch / push 失效 SDK。
- 不实现完整 OTel tracing pipeline。
- 不实现复杂多维 quota / billing / rate limit 后台。
- 不提供 MySQL、Redis 自身高可用部署方案。
- 不做所有缓存的强同步，只做 invalidation hint。
- 不把 memory store 支持为生产级多实例后端。

---

## 3. P0：正确性基线

### 3.1 实例身份与心跳

- 引入稳定 `instance_id`，优先来自 `PROXYHARBOR_INSTANCE_ID`，否则启动时生成。
- MySQL 增加 `instances` 表，记录：
  - `instance_id`
  - `role`
  - `version`
  - `config_fingerprint`
  - `started_at`
  - `last_seen_at`
- 心跳默认每 10s 写入一次。
- 如果启用集群模式，`storage=memory` 必须 fail fast。

### 3.2 轻量 Leader Election

- MySQL 增加 `cluster_locks` 表。
- 使用租约锁实现 leader election：
  - `name`
  - `owner_instance_id`
  - `lease_until`
  - `updated_at`
- 第一版只需要一个锁：`global-maintenance`。
- leader 负责执行全局单例任务，不承担数据面转发特殊职责。

### 3.3 过期 Lease 回收

- leader 每 30s 执行一次过期 lease 回收。
- 每批使用 limit，避免全表大事务。
- 使用全局批量清理接口，不再依赖租户维度调用方。
- 暴露回收日志，指标在后续迭代补齐更多维度。

### 3.4 Lease 乐观锁

- 复用现有 `lease.Generation` 作为乐观锁版本。
- `UpdateLease` 使用 `WHERE tenant_id=? AND lease_id=? AND generation=?`。
- 更新成功后 generation 自增。
- 冲突返回 `stale_lease_generation`，不静默覆盖。

### 3.5 管理操作事务化

- `revoke-all` 等多步 tenant/key 操作应放入一个事务。
- 操作 tenant 行时使用行锁，避免状态变更和 key 新增产生竞态。
- 文档明确：已签发 lease 不承诺被实时 push 失效，新请求必须遵循最新 DB 状态。
- 本轮集成先完成 lease CAS 基线，tenant/key 事务化保留为 v0.3.x 后续补丁项。

---

## 4. P0：跨实例缓存失效

- 将现有 auth invalidator 抽象为公共 invalidation 组件。
- 支持统一事件协议：
  - `tenant.updated`
  - `tenant_key.revoked`
  - `lease.updated`
  - `lease.revoked`
  - `proxy.updated`
  - `provider.updated`
  - `policy.updated`
- Redis Pub/Sub 消息只是 hint，最终一致性来自 DB 读取。
- Redis 不可用时回退到本地 TTL / polling，不影响单实例开发体验。
- 本轮集成保留 auth invalidator，公共 invalidator 作为下一批独立任务，避免和 HA 基线同时大改。

## 5. P1：运行时与云原生

### 5.1 健康写入去重

- 健康 recorder 写入时带 `instance_id` 和时间窗口。
- 第一版采用窗口 upsert，避免副本数增加导致统计成倍放大。
- 后续版本再考虑一致性哈希分片探测。

### 5.2 多实例 selector 默认策略

- 多实例推荐 `redis_zfair`。
- `first` 仅作为 dev / single-instance selector。
- v0.3.0 对多实例 + `first` 先 warning；后续版本可升级为 fail fast。

### 5.3 优雅 Drain

- `/readyz` 应区分：
  - `starting`
  - `serving`
  - `draining`
- 收到 SIGTERM 后：
  - 停止接受新 lease。
  - 等待 inflight 请求结束。
  - 超过 `PROXYHARBOR_SHUTDOWN_TIMEOUT` 后退出。
- Admin 可将 proxy/provider 标记为 draining；新 lease 不再分配，已有 lease 自然过期。

### 5.4 Helm HA 模板

- 默认 `replicaCount` 调整为 2。
- 默认滚动策略：
  - `maxUnavailable: 0`
  - `maxSurge: 1`
- 增加：
  - `PodDisruptionBudget`
  - `terminationGracePeriodSeconds`
  - `preStop`
  - `topologySpreadConstraints`
  - 可选 HPA 模板
- 提供 `charts/proxyharbor/examples/multi-instance-values.yaml`。

## 6. P1：可观测性

### 6.1 Prometheus Metrics

新增 `/metrics`，最小指标集：

- `proxyharbor_instance_up`
- `proxyharbor_leader_active`
- `proxyharbor_inflight_requests`
- `proxyharbor_lease_create_total`
- `proxyharbor_lease_renew_total`
- `proxyharbor_lease_conflict_total`
- `proxyharbor_selector_reject_total`
- `proxyharbor_health_status`
- `proxyharbor_cache_invalidation_total`
- `proxyharbor_cache_stale_seconds`

实例级指标带 `instance_id` label。租户级 label 默认避免进入 Prometheus，防止高基数。
本轮集成先提供 `/metrics` 和 `proxyharbor_instance_up` 骨架，后续指标逐步接入业务路径。

### 6.2 Admin Cluster API

新增 `GET /admin/cluster`，返回：

- 当前实例 ID 和角色。
- leader 锁状态。
- 版本。
- 配置指纹。
- storage backend。
- selector 模式。
- auth refresh 周期。

实例列表和配置漂移聚合依赖 `instances` 查询接口，可作为 v0.3.x 补充。

## 7. P2：后续版本候选

以下内容不进入 v0.3.0 主路径：

- Lease watch / SDK 订阅失效。
- OTel exporter。
- tenant quota。
- proxy max concurrency。
- 完整 Grafana dashboard。
- 更复杂的 health 探测分片。

## 8. 并行开发拆分

### A. Cluster Core

范围：实例身份、心跳、leader election、过期 lease 回收、memory store 多实例 fail fast。
主要文件：`internal/config`、`internal/storage`、`internal/cluster`、`cmd/proxyharbor`、`migrations/mysql/init.sql`。

### B. Correctness & Cache

范围：lease generation 乐观锁、tenant/key revoke-all 事务化、公共 invalidator、lease/proxy/provider/policy cache 失效事件。
主要文件：`internal/storage`、`internal/server`、`internal/auth`、`internal/cache`、`migrations/mysql/init.sql`。

### C. Runtime & Observability

范围：health 窗口 upsert、Prometheus `/metrics`、`/admin/cluster`、request/instance 维度观测字段。
主要文件：`internal/control/health`、`internal/server`、`internal/storage`、`api/openapi.yaml`。

### D. Kubernetes & Docs

范围：Helm HA 模板、PDB/HPA/PreStop/topology spread、multi-instance values、README 和版本文档补充。
主要文件：`charts/proxyharbor`、`README.md`、`README.en.md`、`docs/versions/v0.3.0.md`。

## 10. 验收标准

- 启动两个实例时，只有一个实例持有 `global-maintenance` leader 锁。
- leader 退出后，另一个实例能在锁过期后接管过期 lease 回收。
- 并发 update 同一个 lease 时，冲突不会静默覆盖。
- Redis Pub/Sub 能让 auth cache 最终失效；公共 cache invalidator 后续扩展。
- SIGTERM 后实例遵循 `PROXYHARBOR_SHUTDOWN_TIMEOUT` 做 HTTP graceful shutdown。
- `/metrics` 能返回 Prometheus text 格式。
- `/admin/cluster` 能展示本实例配置摘要和 leader 锁状态。
- Helm 使用 `replicaCount=2` 时具备 PDB、PreStop 和滚动升级安全配置。
