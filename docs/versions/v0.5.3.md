# ProxyHarbor v0.5.3 计划：多实例运维与 Graceful Runtime

> 版本定位：v0.5.3 把多实例从“正确运行”推进到“可运维运行”，补齐 cluster visibility、leader failover 和 graceful shutdown。

---

## 1. 目标

- `/admin/cluster` 展示实例列表、leader、配置摘要和漂移信息。
- heartbeat、leader election、maintenance 单 leader 执行可靠。
- SIGTERM / rolling restart 期间停止接收新 lease，并等待 inflight 请求。
- Kubernetes / Docker 多实例运行状态可观察。

---

## 2. 非目标

- 不提供 MySQL / Redis HA 编排。
- 不实现完整 tracing backend。
- 不做复杂多区域控制面。

---

## 3. P0

- 同一时刻只能有一个 `global-maintenance` leader。
- leader 退出后，其他实例在 TTL 后接管。
- rolling restart 压测期间无明显错误峰值。
- 配置漂移必须可见：storage、selector、redis required、auth invalidation、version。

---

## 4. P1

- Helm HA 模板补齐 PDB、preStop、topology spread、termination grace。
- `/metrics` 暴露 leader、heartbeat、inflight、shutdown/drain 状态。
- Admin API 返回实例 last_seen 与 started_at。

---

## 5. 验收标准

- Docker 3 实例 kill leader 后完成接管。
- 压测期间 rolling restart 成功。
- `/admin/cluster` 能定位配置漂移和 leader 状态。
