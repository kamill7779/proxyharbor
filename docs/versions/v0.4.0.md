# ProxyHarbor v0.4.0 Core Completion

v0.4.0 closes the controller/gateway core loop for dynamic auth, lease lifecycle semantics, audit durability, and Prometheus visibility.

## Scope

### P0 core loop

- Distinguish expired leases from revoked/stale leases with `lease_expired` mapped to HTTP `410` and revoked/stale conflicts mapped to `409`.
- Preserve startup validation through config validation and dynamic auth schema checks.
- Preserve readiness checks for store, cache, selector, and auth-cache dependencies.
- Enforce admin/tenant auth boundaries, including mandatory `X-On-Behalf-Of` for admin calls to tenant-scoped APIs.
- Reject unknown principal types by default instead of silently passing them through tenant-scoped APIs.

### P1 closure

- Expose Prometheus text metrics at admin-protected `/metrics` without tenant labels or other high-cardinality labels.
- Record lease CRUD, gateway validation, selector, auth refresh, health feedback, and audit write failure metrics.
- Persist audit events to MySQL while keeping audit write failures non-blocking and observable through logs and metrics.
- Publish OpenAPI v0.4.0 with `X-On-Behalf-Of`, `reason`, gateway validate, and lease lifecycle error statuses.
- Publish Helm and Compose defaults for cluster, scoring, and zfair configuration.

### Out of scope

Web console, advanced RBAC, quota, rate limiting, policy DSL, reports, alerting rules, tracing backend, and multi-region control plane are not part of v0.4.0.

## Metrics

Business metrics registered by the lightweight internal Prometheus endpoint:

- `proxyharbor_lease_create_total`
- `proxyharbor_lease_create_fail_total`
- `proxyharbor_lease_renew_total`
- `proxyharbor_lease_renew_fail_total`
- `proxyharbor_lease_revoke_total`
- `proxyharbor_lease_revoke_fail_total`
- `proxyharbor_lease_validate_total`
- `proxyharbor_lease_validate_fail_total`
- `proxyharbor_selector_errors_total`
- `proxyharbor_no_healthy_proxy_total`
- `proxyharbor_selector_latency_ms`
- `proxyharbor_auth_refresh_success_total`
- `proxyharbor_auth_refresh_fail_total`
- `proxyharbor_health_events_total`
- `proxyharbor_audit_write_failures_total`
- `proxyharbor_gateway_validate_total`
- `proxyharbor_gateway_validate_fail_total`

The endpoint also emits process/instance gauges: `proxyharbor_process_uptime_seconds` and `proxyharbor_instance_up`.
