# ProxyHarbor v0.5.4 计划：HA 压测、性能门槛与发布收口

> 版本定位：v0.5.4 用实机压力测试收口 v0.5.x 的多实例能力，为 v1.0.0 前的生产可信基线做准备。

---

## 1. 目标

- 固化 HA pressure / soak test 流程。
- 明确性能门槛与可重复测试命令。
- 修复压测暴露的连接池、Redis、MySQL、日志、metrics 开销问题。
- 补齐 README / Helm / 版本文档中 HA 使用说明。

---

## 2. 非目标

- 不把临时压测脚本提交进版本管理。
- 不用测试专门发小版本；测试必须服务于功能发布。
- 不承诺云厂商级大规模 benchmark。

---

## 3. P0

- 3 实例 + MySQL + Redis + LB 的本机 Docker 压测可重复。
- 关键接口 p95 / p99 有记录。
- 压测期间无 panic、无 data race、无连接池耗尽。
- SDK 在 HA 入口下能正常 add proxy / get proxy / renew lease。

---

## 4. P1 初始门槛

- `gateway validate` p95 < 50ms，p99 < 150ms。
- `lease create` p95 < 150ms，p99 < 350ms。
- `lease renew` p95 < 120ms，p99 < 300ms。
- 500 并发、10 分钟 soak test 错误率 < 0.5%。
- MySQL 和 Redis 指标无持续错误。

---

## 5. 验收标准

- HA correctness runner 通过。
- HA pressure runner 达到门槛。
- CI / Docker build / Helm lint 通过。
- README 中只补稳定、验证过的 HA 使用路径。
