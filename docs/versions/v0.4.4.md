# ProxyHarbor v0.4.4: Local selector for SQLite single instance

v0.4.4 keeps the single-instance profile Redis-free by default while preserving Redis zfair for HA deployments.

## Scope

- `selector=local` is the default for `storage=sqlite` single-process deployments.
- The local selector is in-process only; it is not a cross-instance coordination mechanism.
- HA deployments should use MySQL plus Redis with `selector=zfair` and `selector-redis-required=true`.

## Selection behavior

- Local selection uses smooth weighted round-robin.
- Candidates are eligible only when `healthy=true`, `weight > 0`, `health_score > 0`, and `circuit_open_until` is not in the future.
- The selector is concurrency-safe inside one process and avoids pinning all leases to the first available proxy.

## Redis behavior

- `selector=zfair` with `PROXYHARBOR_REDIS_ADDR` configured continues to use Redis zfair.
- `selector=zfair` without Redis falls back to local selection unless `PROXYHARBOR_SELECTOR_REDIS_REQUIRED=true`.
- `PROXYHARBOR_SELECTOR_REDIS_REQUIRED=true` fails startup fast when Redis is missing.

## Metrics

- Selector metrics expose low-cardinality `mode` and `result` labels.
- Proxy IDs and tenant IDs are intentionally excluded from selector metric labels.
