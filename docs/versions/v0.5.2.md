# ProxyHarbor v0.5.2 计划：跨实例缓存失效与状态传播

> 版本定位：v0.5.2 解决多实例状态传播问题，让关键变更不依赖单纯 TTL 等待。Redis Pub/Sub 只作为 hint，数据库仍是最终权威。

---

## 1. 目标

- 抽象公共 invalidation bus。
- 覆盖 tenant、tenant key、lease、proxy、provider 的关键变更。
- Redis Pub/Sub 可用时快速传播，不可用时 TTL / polling 兜底。
- invalidation 不阻塞请求主路径。

---

## 2. 非目标

- 不实现强一致 watch。
- 不要求 SDK 实时订阅失效。
- 不把 Redis Pub/Sub 当作唯一可靠来源。

---

## 3. P0

- 一个实例 revoke tenant key 后，其他实例在可接受时间内拒绝旧 key。
- 一个实例 disable proxy 后，其他实例最终不再分配该 proxy。
- Pub/Sub 丢消息时，DB 轮询能兜底收敛。
- invalidation 消息格式版本化，避免后续扩展破坏兼容。

---

## 4. P1

- `/readyz` 暴露 invalidation 状态。
- `/metrics` 暴露 received、dropped、refresh success/failure、stale seconds。
- 多实例实测包含 Redis 中断与恢复。

---

## 5. 验收标准

- 3 实例下 key revoke / proxy disable 一致性测试通过。
- Redis Pub/Sub 短暂中断后系统恢复。
- 无长期脏读导致的安全问题。
