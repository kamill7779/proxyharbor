## 3. P0：Schema Version 与 Migration 规则

SQLite 必须有 schema version 记录：

```sql
schema_version(version INTEGER NOT NULL, applied_at TEXT NOT NULL)
```

启动时：

- version 缺失：提示运行 init 或自动初始化空库。
- version 过低：按 migration 顺序升级。
- version 过高：拒绝启动，提示 binary 过旧。

migration 规则：

- migration 必须幂等或由 version gate 控制。
- migration 在事务中执行。
- 失败必须回滚并保留原 DB。

## 4. P0：Backup / Restore 安全骨架

提供命令入口：

```bash
proxyharbor backup --sqlite-path ./data/proxyharbor.db --output ./backup/proxyharbor.db
proxyharbor restore --input ./backup/proxyharbor.db --sqlite-path ./data/proxyharbor.db --force
```

安全要求：

- 默认不覆盖已有文件。
- restore 必须显式 `--force`。
- restore 前检查目标目录和 sidecar 文件。
- backup / restore 不打印 secret。
- v0.4.3 可先提供骨架与 guardrail，正式在线备份能力放到 v0.4.5。

---

## 5. P0：Retention Preview

提供 dry-run 预览：

```bash
proxyharbor retention --sqlite-path ./data/proxyharbor.db --audit-days 30 --usage-days 7 --dry-run
```

要求：

- 默认 dry-run。
- 输出将删除的 audit / usage 行数。
- 不删除 tenant、tenant key、provider、proxy、policy、active lease。
- 正式 execute 能力放到 v0.4.5。
