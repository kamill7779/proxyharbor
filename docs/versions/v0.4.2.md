# ProxyHarbor v0.4.2 计划：单体 UX 与本地运维收口

v0.4.2 从 v0.4.0 tag 签出独立开发，目标是围绕“轻量级云原生单体”补齐启动、诊断、初始化和本地运维体验。它可以与 v0.4.1 并发推进，但最终合并时需要在 SQLite 能力之后落地。

## 1. 目标

- 让单体部署从“能跑”变成“容易正确地跑”。
- 提供轻量 `init`/`doctor` 能力，降低配置错误和 schema 错误的排查成本。
- 保持默认路径简单：单体 SQLite；HA MySQL+Redis。
- 不把测试作为单独发布目标；测试随 init/doctor/配置能力一起验收。

## 2. 非目标

- 不做完整 CLI 产品化框架。
- 不做 SQLite 备份/恢复；该能力放到 v0.4.3。
- 不改变 v0.4.0 API 行为。
- 不支持 SQLite 多实例共享状态。

## 3. P0：启动模式简化

### 3.1 单体默认配置

在文档与示例中统一：

```text
role=all
storage=sqlite
selector=local 或无 Redis zfair fallback
```

要求：

- 没有 Redis 时单体可启动。
- 没有 MySQL 时单体可启动。
- 配置错误必须在启动阶段明确失败，不允许 readyz 才暴露根因。

### 3.2 memory 定位降级

`storage=memory` 保留，但只标注为：

- dev
- demo
- CI
- 不持久化

它不作为用户主路径，不在部署矩阵中与 SQLite/MySQL 平级展示。

## 4. P0：Schema 初始化命令

如果 v0.4.1 已经提供自动初始化，本版本补齐显式命令：

```bash
proxyharbor init
```

职责：

- 创建 SQLite 文件。
- 初始化 schema。
- 输出 schema version。
- 不生成 admin key，不写 secret 到日志。

如果 `storage=mysql`：

- 只检查 schema 是否存在。
- 不自动执行 MySQL migration，避免生产误操作。

## 5. P0：Doctor 诊断命令

新增：

```bash
proxyharbor doctor
```

检查项：

- storage driver 是否受支持。
- SQLite 文件路径是否可读写。
- MySQL DSN 是否存在且可连通。
- Redis 是否在 required 时可连通。
- admin key / pepper 是否存在。
- schema version 是否匹配。
- `/readyz` 依赖对应的本地组件是否满足。

输出要求：

- 机器可读的退出码。
- 人类可读的错误提示。
- 不输出 secret 原文。

## 6. P1：Compose 单体体验

Compose 默认示例收敛成：

- 一个 `proxyharbor` 服务。
- 一个本地 volume 保存 SQLite。
- 可选 MySQL+Redis HA compose 放到单独文件或章节。

要求：

- `docker compose up -d` 后 readyz 可用。
- 用户不需要先理解 MySQL/Redis 才能体验核心功能。

## 7. P1：Helm 单副本体验

Helm 默认继续保持云原生约束：

- 默认单副本。
- SQLite 需要 PVC。
- 没有 PVC 时明确提示仅适合临时数据或要求设置 storage。

HA 示例继续使用：

- MySQL。
- Redis required。
- 多副本。

## 8. 并行开发拆分

v0.4.2 可以从 `v0.4.0` tag 签出以下分支并发开发：

### A. Init Command

- 负责 `proxyharbor init`。
- 依赖 v0.4.1 的 SQLite schema 名称；可先用接口占位并在合并时对齐。

### B. Doctor Command

- 负责配置和依赖诊断。
- 不依赖 SQLite 完整实现，可先覆盖 memory/mysql/redis，再补 sqlite 检查。

### C. Compose & README

- 负责单体默认文档。
- 不改运行时代码。

### D. Helm Single Replica

- 负责 values 和模板说明。
- 不引入复杂 Operator 或 CRD。

合并建议顺序：B -> A -> D -> C；若 v0.4.1 未合并，A/C 中 SQLite 细节用 TODO 标注，最终 rebase 后补齐。

## 9. 验收标准

- `go build ./...` 通过。
- `go test ./...` 通过。
- `proxyharbor doctor` 能对 memory/mysql/sqlite 配置给出明确结果。
- `proxyharbor init` 能初始化 SQLite 空库。
- Compose 单体示例可启动并通过 readyz。
- 文档只保留两条正式路径：单机 SQLite、多实例 MySQL+Redis。
