# ProxyHarbor v0.4.7 计划：文档信息架构与单体使用手册

> 版本定位：v0.4.7 不新增运行时能力，专门收口文档结构。目标是把 v0.4.x 已完成的 SQLite 单体、自动 secrets、Go SDK 本地默认路径写清楚，并为 v0.5.x 分布式能力预留目录。

---

## 1. 目标

- 建立 `docs/README.md` 作为文档入口。
- 补齐 Go SDK 文档，按 admin / tenant 权限边界组织。
- 补齐 standalone 部署文档，从启动、配置、架构、数据模型、运维、最佳实践、可观测性展开。
- 增加 distributed 占位文档，明确 MySQL + Redis 是后续 HA 路径。
- 增加 design 文档，沉淀 standalone internals 与 distributed roadmap。
- 保留 `versions/` 作为版本计划入口，并写入 v0.5.x 分布式规划。

---

## 2. 非目标

- 不在本版本实现 v0.5.0 分布式代码。
- 不把未实测的 HA 配置写成用户教程。
- 不改变 README 第一屏风格。
- 不引入临时测试脚本或压测产物。

---

## 3. 验收标准

- `docs/` 目录具备稳定入口和清晰分层。
- Go SDK 文档能从最简路径讲到 admin / tenant / lifecycle / error / recipes。
- 单体部署文档能支撑用户独立完成启动、配置、运维和观测。
- 分布式目录清楚标明占位和后续规划，不误导用户当前能力边界。
- v0.5.0-v0.5.4 规划文件进入版本管理。
