# ProxyHarbor v0.5.1 计划：分布式 Selector 与 zfair 强化

> 版本定位：v0.5.1 在 v0.5.0 写路径正确性之上，强化 MySQL + Redis 多实例下的全局代理选择。代码优先，文档后补。

---

## 1. 目标

- HA 模式明确推荐 `selector=zfair` + `selector-redis-required=true`。
- Redis Lua 选择逻辑具备单元测试、集成测试和多实例实测。
- 多实例并发选择时，全局分配比例接近权重。
- disabled、unhealthy、zero weight、circuit open proxy 不再被新 lease 选择。
- selector 指标可用于定位 no candidate、Redis error、health reject、circuit reject。

---

## 2. 非目标

- 不为 SQLite 单体引入 Redis 依赖。
- 不实现复杂全局最优调度。
- 不引入高基数 proxy_id / tenant_id metrics label。

---

## 3. P0

- zfair 选择必须是 Redis 原子操作。
- HA 模式不允许在 Redis 缺失时静默退回 local selector。
- proxy 状态变化后，selector 不应持续选中不可用 proxy。
- Redis 短暂错误必须返回稳定 domain error，并被 metrics 记录。

---

## 4. P1

- 10 个 proxy、不同权重、3 实例并发 lease create，分配比例在可接受误差内。
- 随机 disable proxy 后，新 lease 不再命中该 proxy。
- Redis 重启场景下，HA strict 模式 fail fast，非 strict 模式行为明确。

---

## 5. 验收标准

- zfair 单元测试覆盖权重、健康、熔断、无候选、Redis error。
- Docker HA selector 压测通过。
- selector 相关 metrics 低基数可观测。
