# ProxyHarbor v0.5.0 计划：多实例写入正确性基线

> 版本定位：v0.5.0 从 v0.4.x 的 single-first 成果，转向 MySQL + Redis 多实例协作。本文档先锁定计划防止丢失；实现以代码、实机验证和压力测试为主，文档在功能稳定后补齐。
> 核心原则：先证明多实例写路径不会错，再谈性能和体验。ProxyHarbor 仍保持轻量云原生控制面，不实现重型一致性协议。

---

## 1. 目标

v0.5.0 要让 2~3 个 ProxyHarbor 实例在同一个 MySQL + Redis 后端上具备以下性质：

- 多实例同时创建、续租、撤销 lease 时不会静默覆盖。
- tenant、tenant key、provider、proxy 等管理写操作具备明确事务边界。
- 幂等请求在多实例负载均衡下返回稳定结果。
- 冲突、重复、已撤销、已过期等语义返回稳定 domain error 与 HTTP status。
- 本机 Docker 能启动完整 HA 测试拓扑，并执行可重复 correctness / pressure 测试。
- 单体 SQLite 路径保持稳定，不被 HA 改造拖复杂。

---

## 2. 非目标

v0.5.0 不做以下内容：

- 不实现 Raft、etcd、Consul 或自研强一致存储。
- 不支持 SQLite 多实例共享状态。
- 不把 memory store 作为生产多实例后端。
- 不在本版本实现 quota、rate limit、策略 DSL、报表或告警。
- 不先写完整用户文档；功能和实测稳定后再补 README / Helm 说明。
- 不提供 MySQL、Redis 自身高可用部署方案。

---

## 3. P0：写路径正确性

### 3.1 Lease 并发语义

- `CreateLease` 必须在多实例下保持 idempotency key 语义。
- `RenewLease` 必须区分 expired、revoked、not found、conflict。
- `RevokeLease` 必须幂等，不能与 renew 竞态后恢复为 active。
- lease generation / version 必须作为乐观锁边界。
- 并发更新冲突必须返回稳定错误，不允许 last-write-wins 静默覆盖。

### 3.2 Tenant 与 Tenant Key 事务化

- tenant disable/delete 与 key revoke-all 必须在一个事务内完成。
- tenant 被禁用后，新 key 创建必须失败。
- key revoke 后，认证缓存最终失效；v0.5.0 至少保证 DB 权威语义正确。
- tenant/key 重复创建返回稳定 conflict。

### 3.3 Provider / Proxy 写语义

- proxy/provider 创建重复返回稳定 conflict。
- disabled / unhealthy / zero weight proxy 不应被新 lease 选择。
- proxy 状态更新必须与 selector 可见状态一致；跨实例缓存刷新可在 v0.5.2 强化。

### 3.4 Error Contract

- 所有 P0 写路径错误都应映射到稳定 domain error。
- HTTP 层继续返回 `reason` 字段，便于 SDK 和压测工具断言。
- 不能让 MySQL driver 原始错误泄漏成 500。

---

## 4. P1：本机 HA 实测环境

### 4.1 Docker 拓扑

提供可重复本机拓扑：

- 3 个 `proxyharbor` 实例。
- 1 个 MySQL。
- 1 个 Redis。
- 1 个 nginx / HAProxy 入口，统一暴露 admin / lease / gateway validate API。
- 固定测试 secrets，避免每次运行脚本人工复制。

### 4.2 Correctness Runner

正式测试工具可以进入 `tools/`；临时脚本不得进入版本管理。

测试维度：

- 并发 lease create / renew / revoke。
- 并发 tenant key create / revoke。
- 并发 proxy add / disable / update。
- idempotency key 多实例重放。
- admin on-behalf-of 与 tenant key 权限边界。

---

## 5. P1：性能门槛

v0.5.0 不追求最终性能，但必须给出可执行门槛，防止正确性修复引入严重退化。

本机 Docker，3 实例 + MySQL + Redis：

- `gateway validate` p95 < 50ms，p99 < 150ms。
- `lease create` p95 < 150ms，p99 < 350ms。
- `lease renew` p95 < 120ms，p99 < 300ms。
- 300 并发、5 分钟 soak test 无 panic、无死锁、错误率 < 0.5%。
- MySQL 连接池无耗尽，Redis selector 无持续超时。

这些数字是 v0.5.0 初始门槛；后续 v0.5.x 可以继续收紧。

---

## 6. 实现顺序

1. 补 HA Docker compose / 本机测试入口。
2. 先写 correctness runner 和关键回归测试。
3. 修 MySQL 写路径事务、冲突、幂等语义。
4. 修 HTTP/domain error contract。
5. 跑多实例 correctness 测试。
6. 跑压力测试并记录瓶颈。
7. 最后补 README / Helm / 版本文档。

---

## 7. 验收标准

- `go test ./...` 通过。
- 本机 Docker HA 拓扑可以一键启动。
- 3 实例下 correctness runner 通过。
- 并发 lease / tenant key / proxy 写入无数据错乱。
- 压力测试达到本文件 P1 门槛。
- v0.4.x 单体 SQLite 快速路径不回退。
