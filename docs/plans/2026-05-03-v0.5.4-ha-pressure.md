# ProxyHarbor v0.5.4 HA Pressure Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a repeatable HA pressure/soak runner, extend the HA SDK verification path, and close the v0.5.4 release surface without widening scope beyond HA pressure, thresholds, and release readiness.

**Architecture:** Keep the existing HA Docker topology and correctness runners as the fixed baseline. Add one focused HA pressure tool under `tools/`, extend `tools/hasdkcheck` for the missing HA lease lifecycle path, and update only the docs/CI surfaces that describe or guard verified HA flows.

**Tech Stack:** Go 1.23, Docker Compose, existing HA tools under `tools/`, GitHub Actions CI, Helm chart docs.
